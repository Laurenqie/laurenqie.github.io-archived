失败也是EXP

所有人一开始都是菜鸟。是在不断的被虐和死亡失败中成长起来。

这是玩PS4联机网战的感受。作为一个娱乐型玩家，其实我打联机很少；自从买了PlayStation Plus，玩了第一把美国末日联机之后就停不下来，准确地说是输的停不下来……

游戏的魅力也在于此：就算你输了，你也会想要继续尝试下一局，因为你得到了良好的反馈（正反馈：虽然我输了，可是我拿了一个人头；负反馈：我死了6次，是因为这里没瞄准……那里没蹲好……我们很清晰地知道自己下一局中可以做出那些改变、提升来做的更好，所以跃跃欲试）

而现在的联机游戏基本都是如此：你开车最后一名，你玩联机死了6次，也一样会有经验值和升级。虽然有时候你觉得：玛德ZZ，明明我没有提升啊？可是当你死着死着，倒数倒数到了几十级，你就发现，诶，我没死那么多了，我开始变成倒数第二名了……

游戏的概念：

失败也是EXP。

在游戏中尤其明显，而在生活中很难感知。正因为在这里我遇到敌人被射死，下次我走到这个地方就会注意，我会越来越熟悉这个地图；正因为这个弯道我没过好，所以我输给了其他车手。那么下一次玩之前我们都会想：到了那个拐角，我应该用什么武器？上一次用了燃烧弹，效果不好，要不这次试试左轮手枪？上一次这个弯道，我刹车踩太多，油门踩太早，要不下一次用手刹？等等……下一次尝试可能还会失败，但是失败就意味着你明白了某种方法行不通，或者某个微操不够好，就会改进……

现实中的任务没有这么详尽的数据，HUD，反馈机制，也比游戏难得多，很多时候我们不知道自己做的到底好不好，大部分时候自知做的很烂，又不知道如何改进，于是失去了继续改进的动力开始拖延……

那么在现实生活中，如何给自己改进的动力和参考呢？我们可以给自己设定数据/升级/等级机制，还可以自行寻找反馈。

比如：

  • 寻找标杆。正如驾驶俱乐部中，计时赛你会看到幽灵车在你前边跑着（那就是你要超越的标杆），跟着它轨迹超过它就行；生活中也是如此，找个榜样，找做的特好的简历和作品集，看着改。

  • 寻找教程/经验。第一次玩TLOU网战，死的太惨，精神崩溃，立马搜索攻略，果然学会不少东西：站起来跑会被敌人听到；用消音的枪可以不被敌人发现；扔出去的钉子炸弹会马上爆炸；打穿防弹衣的敌人要打腿；举枪之前用屏幕中央预瞄……这些东西，如果自己耐心去玩多死几百次可能会发现，但是前人走过弯路留下了经验，你直接汲取，省时省力；工作学习上的事情更是如此，因为他们优先级更高，时间更紧迫……如果你能早点知道这样做是错的，那就省去试错的时间，这就是前人经验的力量。

  • 寻找真人意见。其实打游戏怎么提升最快呀？有个大神——不一定，有个朋友在旁看着，提醒你打枪以后要上弹，靠近战区要蹲下，多听敌人动静……输了以后告诉你刚才后方有敌人等等……你改进起来不就有了现成反馈很意见吗。对的。把你的简历和作品给朋友/前辈看吧！

  • 单机模式。害怕联机？害怕简历投出去马上被刷？先来点单机模式，教程模式，内测模式，把招式练熟了。

  • 最后说一下 等级/经验值机制。在实际工作学习中，我们失败了却感觉不到自己的提升，这是很痛苦的，因为这样只感到挫败而没有成就感，很容易导致不敢继续尝试——最终拖延，影响自己的成长。

以下几个方案可以考虑：

  • 10000小时理论。且不说这个理论到底合不合适。但是时间是最容易也最精准的一种衡量方式了。我们投入在一个工作上的时间是可以衡量的。假设暂定把升到顶级的EXP是10000，那么就朝着这个方向进发——以半个小时的番茄钟为例，你需要完成20000个番茄钟，那么当你看到自己完成了100个番茄钟的时候不管怎么说，都会有成就感的！番茄钟的另一个好处是限定你的工作时间，让你被固定在这段时间里只能工作，这和游戏就相仿了；一局游戏一般是半个小时左右，我们为了最后的游戏结果，通常不会中途退出的；而工作学习就不是这样了，很多时候连半小时不到我们就半途而废，那真正等于“没有结果”（无论是失败还是成功）。
  • 记录。每天记录下来自己做过的事情，收获，失败和成长，回头看笔记时，就知道日子没有虚度。为了每天都能记下来失败/成功经历，你必须去完成任务，如果不挑战，哪来的失败和成长，只会有一纸空白罢了。

非技术岗的互联网从业人员要不要懂技术？

这是个大问题，所以：
此文status：草稿状态。边写边想，欢迎指正。
有待完善例子。

正文开始。

我是个向往技术的人，成天刷些技术论坛，自己也用空闲时间学一学编程。

当然了，很多论调都说“PM不需要懂技术”；也有人说“PM懂技术，流氓会武术”。在我看来，懂技术的好处不在于“哎呀我会编程我能把自己的Idea写成App”！这样固然好，但是对于PM职能来说，是加分项，不是必备项。

但是对于任何互联网从业人员，无论你做的是什么工作，什么岗位，逻辑思维都非常重要。学习计算机科学，可以让人的逻辑思维得到锻炼，更显著的疗效是——因为你理解了你的产品的最终形态（web or App）产生的过程，你才知其所以然。

我现在实习的公司的产品，作为平台+工具型的产品，面对的用户有两端，功能上非常复杂。假如对技术毫无了解，就会出现以下情况：

“打个包就好了。”
“什么是打包？”
“扩展字段是什么？组件是什么？”
“这里可以换图吗？”

这些对话让人精神崩溃。当然，用着用着，新人很快就能打出一个包，生产出一个像样的APP来，但是很有可能，ta还是没明白：”这里为什么不可以换图“。

当然，这些问题，可以通过完善的需求文档名词解释、运营姐姐的耐心辅导解决——可是，你是这个产品的缔造者之一呀，你应该对用户解释，为什么会这样，应该怎么做。如果自己都不理解，那怎么让用户理解？

基本的技术知识，会让你理解”接口“，”数据“，”样式“，以及它们之间的耦合关系，你会知道一个组件显示什么和它背后绑定的数据有关系（有时候，只要知道到这一层，就真的够了）。如果懵懵懂懂只是按照习惯和记忆去完成流程，不代表你真的理解了一个工具。那么下一次产品经理改版，你可能提不出什么意见，或者干脆就是”干啥？干啥玩意儿？“

我觉得，“人人都是产品经理”这句话的意思，不是说让人人都去争这个岗位。这话的意思，是告诫互联网从业者，我们每个人都是我们负责的产品的缔造者，不管是爹妈还是兄弟姐妹，都是有责任的：要理解它，理解它背后的逻辑和运作机制，才能更好地为它的发展做出贡献；而不是囿于自己的职能，设计照着PM的原型图画画儿，开发照着需求文档和设计稿写代码，运营照着新版本的样儿去推广，到头来只有产品经理知道这是咋回事。

以我现有的浅浅的工作经验来看，我发现最会用公司产品的人是运营和测试；前者对界面细节了解的无以复加；后者对技术细节和流程知根知底。产品经理，有时候忙着跟进新需求，对产品现状的某些细节反倒不是那么熟悉（这当然是不对的，也是无奈的）。因此，请教运营和测试，会对自己设计的产品有更深的了解。但是，了解和理解是两码事情。如果一个团队里只有产品经理一个人真正地“理解”产品，这个产品的成长还是会受到很大的限制。

讲过了我的WordPress博客被黑的故事，现在来介绍如何救活它。
网上大把教程都非常复杂，对名词定义又不一，我在这里就说最简单的方法和步骤。

原理：删除了被篡改的WP源码，然后用新的WP源码覆盖之。
风险：你的主题文件和插件以及一些其他附件因此而消失。但是只要你的数据库没有被篡改，文章就都还在。所以，覆盖后一定要登录进WP的后台。主题文件损坏，你的主页是无法显示的。后台地址一般是http://你的域名/wp-admin/
教训：要即时升级WordPress、插件和主题，避免漏洞被利用。密码最好定时更换。同时要经常备份你的数据。

以我的WordPress博客为例，我用的是基于Linux系统的海外主机+CPanel控制面板。

首先登录你的Cpanel

进入文件管理器

把/Public_html下边的所有文件删除。

上传你的最新版，注意，是最新版本的WordPress包到根目录。

解压到根目录。

你的根目录会出现一个WordPress文件夹，

将里边所有文件移到之前你已经清空的Public_html文件夹里。

好了，在浏览器地址栏输入你的域名，看看是不是久违的WordPress设置页出新了？设置好你的数据库，就可以登录WordPress的管理后台了。

这里要注意的是，你的用户名和密码很可能已经被黑客篡改，所以在登录时，选择找回密码，输入你的管理员邮箱，就可以重置密码。

Title: 适合文科生的Markdown编辑器：像前端一样写论文
Date: 2016-02-18 16:44:24
URL: /2016/02/18/markdowneditor/
Tags: markdown , 写论文

#

我一直在找适合文科生的Markdown编辑器，想像前端一样写论文。于是就有了这个总结。

用 Markdown 写作用什么文本编辑器？

这儿为你推荐N款好用的Markdown编辑器。我最喜爱的Mweb和Haroopad，放在最后，可以直接拉到底看。

如果你是一个像我一样喜欢折腾技术折腾各种app的文科生，那么你可能有相似的经历和需求：

在写我的硕士毕业论文（关于D.H.Lawrence的英国文学论文，预估两万字）的过程中，写作的主力工具是换了一个又一个，Evernote，Onenote，Word，Pages……这些富文本编辑器逐渐让我力不从心：

  • 边写论文边用鼠标调格式，没两分钟我就烦了。最后格式一团糟。
  • 在这个编辑器里调好的格式，到另外一个编辑器里就完了。
  • 文章长度超过一万的时候，想要找到其中的一章一节进行修改，我鼠标滚轮都要变成风火轮了。
    于是，我想到了Markdown……
    还记得我在知乎看到某君说，“如果你心情好到要用Markdown来写论文的话。”
    我心情就是那么好。所以——

能写论文的Markdown编辑器需要什么功能？

  • 行号（line number）+ 折叠展开（Markdown Folding）
  • [TOC] 目录/大纲功能
  • 一点都不卡的实时预览
  • 输出Word/PDF/HTML和多样的输出css选择
  • 本地保存或者和印象笔记连接
  • 养眼的主题和自定义样式
    其中，行号和折叠展开、目录功能对写论文至关重要。
    折叠展开是为了写论文的时候可以focus在某一章，把其他的章节都折叠起来。
    目录可以让我跳转到任何一个章节进行修改。提升写论文的效率。
    养眼的主题能让我写得更愉快。输出好看的格式可以直接拿给导师看。一点都不卡的预览才能知道我写的美不美呀~就酱！开始吧

寻找编辑器的旅程

一开始，我是被Workflowy和Dynalist的大纲式文字管理模式给迷住了。OmniOutliner应该也是这样的模式。


Dynalist

在我高高兴兴地把自己的开题报告输进去以后，我发现一个问题……
那就是，输出为word（或者粘贴到word）的时候，那本来很美好的大纲缩进成了悲剧。因为最终要呈现的论文样式，是不能有这样的缩进的。简单来说，我们要递交的论文，其实和纯文本相差不大，只是有标题、段落和脚注而已。

挣扎一段时间以后，我想到，如果Markdown编辑器能够提供折叠标题和大纲导航的功能，那么输出格式的问题不就解决了吗？

要找到这样的东西，我觉得我已经用遍了市面上所有的Web在线Markdown编辑器和Mac、Win系统的MD编辑器……

如果你也是一个对编程有兴趣，有纯文本情节的文科生，要写一篇长长的论文，那么以下是我的推荐：

适合文科生用的Markdown编辑器

1.在线Web app

马克飞象

大名鼎鼎的马克飞象，和Evernote无缝结合，


马克飞象

有在线app和离线chrome app以及系统app，然而它们功能是一模一样的。
有行号+折叠，主题很多而且很美，用起来很顺溜，试用10天内可以和evernote同步。
一开始我是真想，就它算了，虽然它在本地不能那么方便的保存，也不能打开本地文件……
然而一年要79rmb，不是一次性购买，要年年付款啊，我……选择放弃。

作业部落CMD Markdown

带有大纲功能！可以调转到你的各个章节很方便啊！同样可以同步到evernote。免费。
没有行号，没有折叠。
然而，主题真的不适合我。字还特大。还不能调。心酸。


CMD MARKDOWN

小书匠

有行号，有目录
设置非常丰富，可以自己自定义css，很棒！
感觉是国内做的在线MD编辑器功能最丰富的。


小书匠

国外的编辑器也有很多，比如StackEdit,我感觉还是Dillinger最好用，可以把MD文件同步到Onedrive。界面很清爽。不能换主题。

Dillinger

2.Mac的Markdown编辑器

作为Mac党，还是想要一款离线的本地App，可以随开随用的。就算没有同步功能，但是我可以存到本地，然后同步云盘嘛。

著名的Mou和MacDown都是很好的MD编辑器，我没有用的原因是，它们太太太简约了，在编辑长文章的时候让人困扰。IA Writer和Typora这类极端简约（不带预览）的编辑器其实也很棒，如果你是想要专心写作，不像我要求丰富，完全可以尝试以上提到的轻量Markdown编辑器。


Typora

我的博客和网站都是基于Wordpress建立的，图床也是采用的七牛云存储，结果真的有这样一款app:Mweb,可以满足这些需求：

  • 把文章发布到Wordpress（虽然我还没会……
  • 把本地图片自动上传到七牛云

MWeb

虽然没有我心心念念的行号的折叠功能，但是发布功能很强大，又看到App作者的故事，我还是义无反顾地买下了Pro版，68RMB。

Mweb

三栏的视图和大部分笔记软件很接近，可以用它代替笔记软件，建立自己的笔记系统和静态网站。视图功能很丰富，几乎满足使用Markdown的人的一切需求。

虽然为知笔记也支持Markdown，可是为什么每次打开，都卡的不得了，而且文件还会丢失，心很痛，最后还是放弃了为知笔记。

最良心的是作者一直在更新一直在更新。不像某些跳票的App作者。而且发邮件真的会回复，我这个用户的各种奇葩问题。23333

现在成为我的Wordpress文章主力编辑工具。自动传图和完美的功能，简直棒呆！

相似Mac软件推荐：

Ulysses 和 Quiver 都是Markdown语法的笔记软件。这意味着双栏实时预览不是主打功能，可能不是很好用。但是，做笔记是一流的，如果你不是需要最终将写作内容输出成完美格式，拿它们来做笔记也是甚好的！

HaroodPad


Haroopad

这是一个韩国开发团队的作品，它已经成为了我的论文主力编辑器。

成功地满足了我的需求：

  • 行号+折叠+TOC目录
  • 丰富的主题+不卡的实时预览
  • 支持本地文件存储和读取
  • 输出的文件简直不要太美惹

虽然还是有一些缺憾，比如不能和各种网盘无缝衔接，没有字体调整（得自己去写css）等等，但是已经近乎完美。写论文没错儿！

另外

假如你是码农，而不是我这种对代码一知半解的文科生，
完全可以用你滴工作编辑器来打造Markdown编辑器，
比如Sublime Text装上Markdown插件，
或者Atom的各种Markdown+Folding+七牛package，应有尽有。
以上我都试过，然而我还需要继续学习。多多指教！
